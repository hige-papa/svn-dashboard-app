# 🚀 未経験から始めるNuxt 3 + Vue 3入門！モダンなWeb開発の第一歩

「モダンなWebサイトを作ってみたいけど、何から始めればいいかわからない…」
そんなあなたに、**Nuxt 3**と**Vue 3**を使った開発の始め方を、わかりやすく解説します！

Nuxt 3は、Vue 3をベースにしたフレームワークで、**直感的**かつ**パワフル**にWebアプリケーションを構築できます。この記事を読み終える頃には、あなたもNuxt 3での開発の第一歩を踏み出せているはずです。さあ、一緒に始めましょう！

## 1. 準備をしよう 💻：開発環境のセットアップ

まずは、開発に必要な道具を揃えましょう。

### **Node.jsのインストール**

Nuxt 3はNode.jsという環境の上で動作します。公式サイトから**LTS版**（推奨版）をダウンロードして、インストーラーの指示に従ってインストールしてください。

[➡️ **Node.js公式サイト**](https://nodejs.org/)

インストールできたら、ターミナル（WindowsならコマンドプロンプトやPowerShell、Macならターミナル）を開いて、以下のコマンドを打ち、バージョンが表示されれば成功です。

```bash
node -v
npm -v
```

### **Visual Studio Code (VS Code) の導入**

次に、コードを書くためのエディタをインストールします。VS Codeは無料で高機能、多くのWeb開発者に愛用されています。

[➡️ **VS Code公式サイト**](https://code.visualstudio.com/)

インストールしたら、さらに開発を便利にするための**拡張機能**も入れておきましょう。

  * **Volar (Vue Language Features)**: Vue 3とTypeScriptの強力なサポートを提供します。これがないと始まりません。
  * **ESLint**: コードの品質を保ち、エラーを未然に防ぎます。
  * **Prettier - Code formatter**: コードを自動で綺麗に整形してくれます。

-----

## 2. プロジェクトの誕生 🎉：Nuxt 3プロジェクトを作成する

環境が整ったら、いよいよプロジェクトを作成します。

ターミナルを開き、プロジェクトを作成したいディレクトリに移動して、以下のコマンドを実行してください。`<project-name>`の部分は、あなたの好きなプロジェクト名に変えてくださいね。

```bash
npx nuxi@latest init <project-name>
```

コマンドを実行すると、いくつか質問されるかもしれませんが、基本的にはEnterキーで進めて大丈夫です。

プロジェクトが作成されたら、そのディレクトリに移動して、必要なパッケージをインストールします。

```bash
cd <project-name>
npm install
```

これで準備は万端です！

-----

## 3. "Hello, Nuxt\!" 👋：開発サーバーを起動する

早速、作成したプロジェクトを動かしてみましょう。以下のコマンドを実行してください。

```bash
npm run dev
```

ターミナルに `Local: http://localhost:3000/` のようなURLが表示されたら成功です。このURLをブラウザで開いてみてください。「Welcome to Nuxt\!」という画面が表示されるはずです。

おめでとうございます！これがあなたの最初のNuxtアプリケーションです。

-----

## 4. 作ってみよう 🛠️：基本的な開発の流れ

ここからは、実際にコードを書きながら、Nuxt 3の基本的な開発スタイルを体験していきましょう。

### **ページの作成（ファイルベースルーティング）**

Nuxt 3の大きな特徴の一つが、**ファイルベースルーティング**です。`pages`ディレクトリに`.vue`ファイルを作成するだけで、自動的にその名前のURL（ルート）が作成されます。

1.  プロジェクトのルートに`pages`という名前のディレクトリを作成します。

2.  `pages`ディレクトリの中に`about.vue`というファイルを作成し、以下のコードを貼り付けてください。

    ```vue
    <template>
      <div>
        <h1>Aboutページです</h1>
        <p>Nuxt 3の世界へようこそ！</p>
        <NuxtLink to="/">トップページへ</NuxtLink>
      </div>
    </template>
    ```

これで、ブラウザで `http://localhost:3000/about` にアクセスしてみてください。先ほど作成したページが表示されるはずです。簡単ですよね？

### **コンポーネントの作成（Auto-import）**

次に、繰り返し使えるUIパーツである**コンポーネント**を作ってみましょう。Nuxt 3では、`components`ディレクトリに作成したコンポーネントは、どこでも**インポート文なしで**利用できます（**Auto-import**）。

1.  プロジェクトのルートに`components`という名前のディレクトリを作成します。

2.  `components`ディレクトリの中に`TheHeader.vue`というファイルを作成し、以下のコードを貼り付けます。

    ```vue
    <template>
      <header style="background-color: #f0f0f0; padding: 1rem;">
        <h2>これは共通ヘッダーです</h2>
      </header>
    </template>
    ```

3.  `app.vue`（プロジェクトのルートにあるファイル）を以下のように書き換えて、ヘッダーコンポーネントを使ってみましょう。

    ```vue
    <template>
      <div>
        <TheHeader />
        <NuxtPage />
      </div>
    </template>
    ```

    `<NuxtPage />`は、現在のURLに対応する`pages`ディレクトリ内のコンポーネントを表示するための特別なコンポーネントです。

これで、どのページにアクセスしても共通のヘッダーが表示されるようになりました。

### **データの取得 (`useFetch`)**

Webサイトには、外部からデータを取ってきて表示する機能が欠かせません。Nuxt 3では`useFetch`という便利な関数が用意されています。

`pages/index.vue`というファイルを作成（もしなければ）して、以下のコードを試してみましょう。これは、外部のテスト用APIからユーザーリストを取得して表示する例です。

```vue
<template>
  <div>
    <h1>ユーザーリスト</h1>
    <div v-if="pending">読み込み中...</div>
    <ul v-else>
      <li v-for="user in users" :key="user.id">
        {{ user.name }}
      </li>
    </ul>
  </div>
</template>

<script setup>
const { data: users, pending } = await useFetch('https://jsonplaceholder.typicode.com/users');
</script>
```

`script setup`の中で`useFetch`を使うだけで、非同期でのデータ取得と、その状態（`pending`：読み込み中か否か）の管理が簡単にできてしまいます。

## 5. 次のステップへ 📖

ここまでで、Nuxt 3開発の基本的な流れは掴めたはずです。さらに学びを深めるためのキーワードをいくつか紹介します。

  * **レイアウト (`layouts`):** 複数ページで共通の骨組み（ヘッダー、フッターなど）を使いたい場合に便利です。
  * **状態管理 (Pinia):** アプリケーション全体で共有したいデータを管理するための仕組みです。
  * **Nuxt Modules:** UIライブラリや認証機能など、様々な機能を簡単に追加できます。
  * **ビルドとデプロイ:** `npm run build` や `npm run generate` コマンドで、作成したサイトを公開できる形にします。

これらの機能については、ぜひ[**Nuxt 3の公式ドキュメント**](https://nuxt.com/docs/getting-started/introduction)（日本語にも対応しています）を読んでみてください。

この記事が、あなたのモダンWeb開発者としてのキャリアの素晴らしいスタートになることを願っています。Happy Coding\! 💻✨
